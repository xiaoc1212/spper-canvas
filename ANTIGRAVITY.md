# 超级画板 (ConfigVault) - 项目核心开发指南

这是针对 AI 编程助手（Antigravity / Claude 等）的项目核心记忆指南。每次开始新任务介入前，请**务必首先阅读此文件**以建立对项目的正确认知。

## 1. 基础运行环境
*   **前端框架**: React + Vite + TypeScript.
*   **开发端口**: **3000** （不是 5173！详见 `vite.config.ts`）。
*   **启动命令**: `npm run dev` (通常在终端里保持运行)。

## 2. 架构设计与状态管理
*   **核心组件**: 画布的交互核心均在 `components/ContentArea.tsx` 内进行。
*   **技术栈**:
    *   **动画与手势交互**: `framer-motion`。
    *   **UI 与样式**: Tailwind CSS。在修改任何组件时优先使用 Tailwind className。
    *   **图标库**: `lucide-react`。

## 3. 设计规范与审美限制
*   **UI 风格**: 极客风、原生桌面级交互体验。倾向于类似 Milanote、Notion 融合的高级暗色系外观。
*   **交互要求**: UI 要有呼吸感和“物理真实性”（如微动画过渡、毛玻璃、精细边框亮面），在交互时绝对不允许出现原生网页带来的粗糙感（比如：框选时导致的乱码文本选取应被 `select-none` 合理封锁）。

## 4. 画布操作“宪法”总结 (避免破坏)
*   **极限尺寸**: 物理容器目前定义为 `CANVAS_SIZE = 15000` 像素极大幅度。这也是为了确保无论怎么放缩、挪拽都不出边界而特意设计的，不要轻易去调整它改回去。
*   **缩放系统**: 采用 React 的原生事件（mousedown, pointermove 等）作为驱动。特别注意：**缩放倍率必须使用 `useRef` (即 `canvasScaleRef`) 追踪进入原生 Event Listener**，否则会导致经典的 React 过期闭包 (Stale Closure) 陷阱，算错偏移坐标。
*   **边界系统**: 画板使用了 `scrollLeft` 加上中心化锚定缩放。卡片的移动被限制在这个画板系统内。
